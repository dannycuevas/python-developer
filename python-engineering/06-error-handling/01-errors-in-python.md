- An "error" that crashes our code when executing, is called an "exception"  
- Python raises these "exceptions" whenever the interpreter does not understand what to execute, like telling us humans something like "I have no idea what you are doing, I am going to stop executing the code and give you back an output"
- **How can we handle these "exceptions" that crash our code when running? we use something called "error handling"**

### Error Handling

- A natural part of a great programmer, is able to handle errors in their program
- **"Error Handling" allows us to let a Python script continue running even if there are any errors, that is the magic right there**

### Different types of errors

- `TypeError` = incompatible different data types
- `SyntaxError` = not using correct Python syntax
- `IndexError` = index does not exist 
- `KeyError` = accessing something in a Dictionary that does not exist
